  • 如何看待前端框架选型？
    • 前端框架选择要根据项目和团队实际情况来考虑，目的是高效的开发项目。可以从这几个角度考虑：
      • 这个框架的使用场景。
      • 团队当下的技术能力。
      • 框架能解决哪些问题。
      • 框架的生态系统。
      • 跨平台性。
    • 当前主流的三个框架：Angular、Vue、React。
      • Angular：它是一个成熟完善的框架，而React和Vue只是一个UI组件库。angular适用于大型项目，学习成本高。
      • Vue：vue一开始的定位就是尽可能的降低前端开发的门槛，让更多的人能够更快地上手开发，类似全家桶都搭建好了。
      • React：React 一开始的定位就是提出 UI 开发的新思路，只关注视图层，因为强大的生态圈可以按照自己的实际情况来搭建项目组合。
    • 总结
      • 假如你想快速开发一个简单学习成本低的小应用，可以考虑Vue。
      • 假如你想开发一个大型应用，可以考虑angular。
      • 假如你想开发一个跨平台应用，可以考虑react。
  • vue的如何实现双向绑定的？
    • 使用浏览器提供的订阅-发布的功能（API：observe-subscribe）。
    • defineProperty来截获属性的变化。
  • react 虚拟DOM 是什么? 如何实现? 说一下diff算法。
    • 虚拟DOM：render函数中返回的element的集合，是对DOM树的描述。
    • 实现：根据jsx结构，使用createDocumentFragment创建对应的虚拟的节点对象。
    • diff算法：先内存中保存两份虚拟DOM，DOM的每一层结构有唯一的key值（如果不主动设置，react会根据规则提供一个默认的），在render时，会深度遍历虚拟dom节点，将两个虚拟dom同层比较，如果key值相同，标识无变更，不作处理；如果key值不同，则进行修改（删除或添加），并放入dirtyQueue（脏队列或者叫变更缓存队列，用来存储更新后的结构）等到变更检测结束之后，从dirtyQueue里面取出新的结构并更新。
